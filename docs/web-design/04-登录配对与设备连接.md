# 登录、配对与设备连接

## 1. 页面

登录页、首次引导、设备配对、设备授权、登录设备管理、会话过期页。

## 2. 登录流程

1. 用户使用账户或组织身份登录。
2. 服务端校验后进入设备选择；没有设备时进入配对引导。
3. 用户在桌面端生成短期配对码或二维码。
4. Web 输入配对码，桌面端显示浏览器、位置和权限申请。
5. 桌面端确认后建立设备信任，Web 拉取能力快照。

## 3. 权限等级

| 权限 | 能力 |
|---|---|
| 查看 | 历史、分析、设备和文件元数据 |
| 操作 | 新建对话、读取文件、运行安全工具 |
| 代码写入 | 修改文件、Git 暂存、Worktree 操作 |
| 管理 | 供应商、Hook、备份、设备撤销 |

权限可按设备或项目收窄。高风险能力默认不随基础配对自动授予。

## 4. 设备状态

在线、执行中、待审批、连接不稳定、离线、版本过低、权限受限、需要重新配对。状态卡展示最后心跳、应用版本、平台、当前任务、同步时间和可用能力。

## 5. 异常处理

- 配对码错误：保留输入并显示剩余尝试次数。
- 配对码过期：清空并提供重新生成入口。
- 桌面拒绝：说明未建立信任，不自动重试。
- 网络中断：进入等待状态，恢复后继续握手。
- 版本不兼容：只允许查看兼容数据并提示升级桌面端。
- 会话过期：保存未发送输入，登录后恢复。

## 6. 安全细节

- 配对码短期有效且单次使用。
- 设备授权页面必须显示浏览器、系统、时间和大致来源。
- 支持撤销单个浏览器、全部 Web 会话或整台设备。
- 敏感操作可要求再次验证。
- 页面永不显示 API Key、Hook Token 或系统凭据明文。

## 7. 移动端

配对优先使用扫码；授权列表使用单栏卡片。危险撤销操作放在页面底部危险区，避免误触。
