# 状态矩阵与验收标准

## 1. 通用页面状态

| 状态 | 必须表现 |
|---|---|
| 首次加载 | 骨架与模块标题，避免全屏 Spinner |
| 局部刷新 | 保留旧数据，显示刷新状态 |
| 空数据 | 原因、下一步和权限说明 |
| 部分失败 | 正常区域继续可用，失败卡片可重试 |
| 全局失败 | 错误编号、重试、返回和诊断入口 |
| 离线 | 最后同步时间、只读标识、禁用写操作 |
| 缓存可用 | 显示缓存标识、桌面来源和最后上报时间 |
| 缓存过期 | 保留只读内容，显示过期警告并禁止写操作 |
| 无权限 | 说明所需权限和申请入口 |
| 版本不兼容 | 可用能力降级和升级提示 |

## 2. 写操作状态

准备、提交中、等待桌面、等待审批、执行中、成功、失败、冲突、超时、取消。所有状态显示同一个操作 ID，防止用户重复点击产生多个任务。

## 3. 场景矩阵

| 维度 | 场景 |
|---|---|
| 设备 | 在线、连接不稳定、离线、版本过低 |
| 项目 | 主仓库、Worktree、SSH、路径缺失 |
| CLI | Claude、Codex、未安装、Hook 未安装 |
| 会话 | 新建、运行、待审批、完成、失败、恢复 |
| 浏览器 | 桌面、平板、移动端、后台标签页 |
| 网络 | 正常、高延迟、断开、重连、事件缺口 |
| 权限 | 查看、操作、写代码、管理、权限撤销 |

桌面端原有窗口焦点、分屏、最小化、托盘、Shell、WSL、Worktree 和 Hook 安装状态仍由桌面代理处理，并通过结构化状态反馈 Web。

## 4. 功能验收

- 用户能确认当前设备、项目、Worktree、分支和 CLI。
- 所有桌面执行请求有回执，重复提交不会重复执行。
- 所有业务数据都能追溯到桌面设备、上报时间和版本。
- Web 缓存不被当作新的业务数据源，桌面最新快照可以覆盖缓存。
- 操作在桌面回执前只能显示“已提交、等待桌面或执行中”。
- 审批卡完整显示风险和影响范围。
- 文件保存和配置写入具备版本冲突保护。
- 历史、Replay、Diff 能双向定位。
- 图表支持下钻、键盘和数据表。
- 敏感字段永不回显明文。
- 离线状态不会出现可执行主按钮。
- 设备离线时只允许查看缓存和编辑本地草稿，不允许排队执行危险操作。
- Rust 服务重启后能从 SQLite3 和桌面快照恢复，进程内缓存丢失不影响业务正确性。
- 单实例运行不依赖 Redis；未满足引入条件时部署清单中不存在 Redis。
- SQLite3 开启 WAL、外键和 busy timeout；高频事件使用批量短事务，不能长期占用写锁。
- 数据库在线备份和恢复经过完整性检查，不直接复制正在写入的数据库文件。
- 同一 SQLite3 数据文件只允许一个 Rust 服务实例管理；需要水平扩容时先迁移数据库。
- 移动 Web 冷启动、后台恢复和断线重连均先展示最后有效快照，再无闪烁刷新。
- 通知和分享链接能定位到正确设备、项目、Worktree 和会话。

## 5. 视觉验收

- 组件使用统一 Token、间距、圆角和图标体系。
- 不使用 Emoji 作为功能图标。
- 桌面页面无无意义大留白，移动端无整页横向滚动。
- 代码和 Diff 仅组件内部横向滚动。
- 中文和英文长文本不破坏布局。

## 6. 无障碍验收

- 键盘可到达全部交互控件。
- Dialog 和 Drawer 正确管理焦点并支持 Escape。
- 图标按钮具有可访问名称。
- 状态变化通过可感知文本播报。
- 颜色对比、目标尺寸和减少动画符合规范。

## 7. 安全验收

- 配对码短期、单次有效。
- 写操作校验用户、设备、项目和目标版本。
- 路径操作由桌面端 canonicalize 并校验项目边界。
- 密钥、密码、Token、私钥和完整环境变量不进入 Web。
- 审计覆盖审批、写文件、Git、Worktree、Hook、供应商和恢复。
- 退出登录、撤销设备或降低权限后，相关服务端和浏览器缓存立即失效。

## 8. 交付检查

- 文档链接和图片路径有效。
- 原型覆盖主要页面和关键异常状态。
- 产品代码、数据库和构建配置无修改。
- `git diff --check` 通过。
- 后端架构和移动 Web 调研文档已纳入总索引。
