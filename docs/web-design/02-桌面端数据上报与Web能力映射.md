# 桌面端数据上报与 Web 能力映射

## 1. 数据原则

- 桌面端是所有业务数据、运行状态和操作结果的唯一可信来源。
- Web 端不直接访问项目文件、Git、CLI、Hook、系统资源、SQLite 或本地历史目录。
- Web 发起的操作必须下发到指定桌面设备，由桌面执行后上报过程和结果。
- Web 服务只缓存桌面上报的结构化数据，不独立生成或修改业务事实。
- Web 缓存不保存 PTY 原始流和敏感明文。
- 快照用于首次加载，事件用于实时更新，回执用于确认写操作结果。
- 每条事件包含 `deviceId`、`projectId`、`sessionId`、`sequence`、`occurredAt`。
- 所有写请求包含幂等键、目标版本和过期时间。

## 2. 操作链路

```text
Web 用户操作
  → Web 生成 operationId 和请求摘要
  → 服务端鉴权并转发到目标桌面
  → 桌面校验设备、项目、路径、权限和版本
  → 桌面执行真实操作
  → 桌面持续上报进度事件
  → 桌面上报最终回执和最新状态
  → Web 更新缓存并刷新界面
```

- Web 可以立即显示“已提交”，但不能提前显示“成功”。
- 桌面拒绝、超时、离线或版本冲突时，Web 必须保持原业务状态。
- 文件写入、Git、Worktree、Hook、供应商和备份操作不做离线执行队列。
- 只允许草稿、筛选条件和未发送 Prompt 在浏览器本地保存。

WebSocket 网关、业务编排、缓存和 API 均由同一个 Rust 服务提供。连接层只维护连接与转发，业务层只处理权限、状态机和缓存，二者通过内部模块接口协作。

## 3. 上报通道

| 通道 | 频率 | 内容 |
|---|---:|---|
| 心跳 | 10–30 秒 | 在线状态、版本、能力、当前任务数 |
| 状态快照 | 登录、重连、主动刷新 | 项目、会话、任务、配置摘要 |
| 实时事件 | 即时 | Prompt、工具、文件变更、审批、通知 |
| 指标批次 | 30–60 秒 | CPU、内存、Token、费用、网络 |
| 历史增量 | 文件变化后 | 会话索引、消息、usage、Diff 引用 |
| 命令回执 | 操作结束 | 成功、失败、冲突、拒绝、超时 |

## 4. 功能映射

| 桌面能力 | Web 表达 | 数据或执行位置 |
|---|---|---|
| 项目树、分组、健康检查 | 项目列表和状态卡 | 桌面上报；编辑由桌面落库 |
| PTY、Tab、分屏 | 结构化对话和任务列表 | 桌面执行，Web 不展示终端 |
| Claude/Codex Hook | 对话时间线、审批、通知 | Hook 事件上报 |
| 文件浏览和搜索 | 文件树、搜索、预览 | 桌面按需读取，内容短期缓存 |
| Git 变更、分支、提交 | Git 工作区 | 桌面执行 Git 并回传结果 |
| Worktree | 并行任务工作区 | 桌面创建、删除、校验 |
| 历史会话 | 历史工作区 | 桌面索引后增量同步 |
| AI Replay | 结构化轮次、工具、快照 | 桌面解析，Web 展示 |
| 用量和请求日志 | 分析与日志 | 聚合数据上报 |
| 供应商切换 | 项目供应商设置 | 密钥留在桌面端 |
| Hook/状态栏 | 状态和配置请求 | 桌面检测、安装、写配置 |
| WebDAV 备份 | 版本列表和恢复任务 | 桌面持有凭据并执行 |
| 系统资源 | 设备监控 | 只读指标上报 |
| 桌宠、托盘、窗口、终端主题 | 桌面专属说明 | 不在 Web 复刻 |

## 5. Web 缓存模型

### 5.1 缓存用途

- 加快首次加载和页面切换。
- 设备短暂断线时保留只读内容。
- 支持历史、分析和请求日志分页。
- 减少相同快照和静态元数据的重复上报。

### 5.2 可缓存内容

| 数据 | 缓存方式 | 失效条件 |
|---|---|---|
| 设备和能力快照 | 短期缓存 | 心跳超时、版本变化、设备撤销 |
| 项目、分组、Worktree 索引 | 版本缓存 | 桌面上报新版本或路径状态变化 |
| 会话和历史索引 | 增量缓存 | 新事件、删除、来源目录切换 |
| Replay、Prompt、请求日志 | 分页缓存 | 桌面增量序号变化 |
| 统计聚合 | 时间范围缓存 | 新 usage 上报、价格变化、主动刷新 |
| 文件树和 Git 状态 | 短期缓存 | watcher/Git 事件、分支或 Worktree 变化 |
| 文件正文和 Diff | 限时缓存 | 文件哈希变化、会话结束、权限撤销 |

### 5.3 禁止缓存内容

供应商密钥、WebDAV 密码、Hook Token、SSH 私钥、完整环境变量值、系统凭据、PTY 原始输出、未授权文件正文和桌面临时授权信息。

### 5.4 缓存字段

每条缓存至少保存 `deviceId`、资源类型、资源 ID、桌面版本号、`sequence`、`reportedAt`、`cachedAt`、过期时间和内容哈希。页面必须能判断数据是实时、缓存、过期还是设备已撤销。

### 5.5 缓存层级

- 服务端缓存：写入 SQLite3，保存经过授权的桌面上报数据，供跨浏览器和移动端读取。
- 浏览器内存缓存：减少同一会话内重复请求。
- 浏览器持久缓存：只保存主题、布局、草稿、筛选和经过允许的只读快照；退出登录或撤销设备后清理。

首版不依赖 Redis。Rust 进程内缓存只保存可重建的热点数据；SQLite3 保存跨重启缓存。SQLite3 采用 WAL、外键、busy timeout 和短事务；所有写入经统一存储模块排队，避免多个业务模块竞争写锁。

## 6. 敏感数据

禁止进入 WebView 或 Web 服务：供应商密钥、WebDAV 密码、Hook bearer token、完整环境变量值、系统凭据、未授权文件内容。Web 只展示脱敏尾号、配置是否存在及验证状态。

## 7. 离线和一致性

- 心跳超时后设备转为“连接不稳定”，连续超时转为“离线”。
- 离线设备允许查看最后同步数据，但禁用写操作。
- 页面显示最后同步时间和数据新鲜度。
- 版本冲突时拒绝覆盖，要求刷新后重试。
- 重连后以 `sequence` 补传事件；缺口过大时重新拉取快照。
- 桌面新快照始终覆盖同版本之前的 Web 缓存。
- 收到乱序或重复事件时按 `sequence` 去重，不回退到旧状态。
- 缓存过期后仍可只读展示，但必须显示“数据可能已过期”。
- 设备撤销、用户退出或权限降低后立即清除相关缓存。

## 8. 文件与历史存储策略

- 文件正文默认按需读取，不做长期云端镜像。
- Diff 可保存结构化摘要和短期内容引用。
- 历史消息可按用户策略同步；敏感项目支持仅索引元数据。
- Replay 快照只同步元数据和经过授权的 Diff，不上传整个仓库。

## 9. 操作审计

记录操作者、设备、项目、Worktree、操作类型、参数摘要、审批结果、执行结果和时间。命令、路径及 Diff 按权限脱敏，审计记录不可由普通用户修改。
