# 项目与 Worktree

## 1. 项目列表

支持分组、搜索、来源设备、CLI 类型、健康状态、收藏和最近访问筛选。卡片展示名称、路径摘要、设备、分支、未提交变更、运行任务、今日 Token 和最后活动。

## 2. 项目详情

页签包括概览、会话、Worktree、文件、Git、统计和设置。概览展示项目健康、CLI 可用性、Hook、供应商、当前分支、活跃任务及最近会话。

## 3. 项目操作

新建、编辑名称、移动分组、收藏、选择 CLI、选择 Shell、配置启动命令、切换供应商、打开桌面目录、归档和删除。Web 新建项目只允许选择桌面端上报的可访问目录，不能输入任意服务器路径绕过边界。

## 4. Worktree

- 列表显示分支、路径、设备、状态、任务、变更量和创建时间。
- 创建时选择基准分支、新分支名、目录建议和是否立即启动任务。
- 删除前检查未提交变更、运行任务和目录存在性。
- 主项目与 active Worktree 的今日用量展示同一项目聚合，同时保留单 Worktree 明细。

## 5. SSH 项目

显示主机、远程路径、连接状态和主机指纹摘要。Web 不保存 SSH 私钥；连接由桌面端现有 SSH 配置执行。断线时展示最后状态，并允许桌面代理重连。

## 6. 健康状态

正常、目录缺失、设备离线、Git 不可用、CLI 缺失、Hook 未安装、供应商异常、Worktree 缺失、SSH 不可达。每个状态提供明确修复入口或说明必须在桌面端处理。

## 7. 冲突与权限

项目编辑采用版本校验。路径、CLI、供应商和环境变量变更属于管理操作；环境变量只显示键名和脱敏值，修改值后不可再次读取明文。

## 8. 移动端

项目列表使用卡片；详情页签改横向可滚动标签。创建 Worktree 使用分步表单，危险删除按钮与常规操作分区。
