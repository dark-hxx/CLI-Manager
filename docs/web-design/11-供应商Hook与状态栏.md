# 供应商、Hook 与状态栏

## 1. 供应商列表

按 Claude、Codex 分类显示名称、BASE_URL 主机、模型、状态、来源和使用项目数。密钥由 Rust 侧脱敏，Web 只看到是否已配置和尾号摘要。

## 2. 项目级切换

项目可选择跟随全局或指定供应商。页面显示合并优先级、当前生效来源、`settings.local.json` 冲突警告和下次会话生效提示。切换请求由桌面端生成 Claude `--settings` 或 Codex `--profile`。

## 3. 跨环境

标明配置位于 Windows、WSL、macOS 或 Linux。WSL 数据库通过桌面端安全备份读取和事务更新；运行时缺失或并发冲突时返回可诊断错误，不自动安装依赖。

## 4. Hook 管理

分别展示 Claude/Codex Hook 的安装状态、目录、版本、事件、通知和最近测试。操作包括检测、安装、修复、测试和卸载；安装与卸载属于管理权限并要求确认。

## 5. Hook 通知

配置审批、完成、失败等事件；目标可包含系统通知及已支持的第三方平台。凭据字段只允许重新输入或清除，不支持复制明文。

## 6. Claude 状态栏

三行画布、组件库、搜索、拖拽、属性编辑、颜色、粗体、自定义文本/命令、Powerline 和实时预览。支持多份命名配置、导入旧配置、检测、安装和卸载。

## 7. Codex 状态栏

选择模型、上下文、额度、Git、Token、权限、会话和任务进度，拖拽排序并预览。保存仅更新 `[tui].status_line`，其他配置由桌面端保留。

## 8. 错误与回滚

配置解析失败、路径不可写、目标环境缺失、旧值冲突、测试失败。写入采用临时文件和替换；失败保留备份并提供恢复入口。

## 9. 移动端

供应商和 Hook 适合查看与切换；复杂状态栏设计器在移动端只读预览，并提示使用桌面或大屏浏览器编辑。
