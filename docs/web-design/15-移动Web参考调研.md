# 移动 Web 参考调研

## 1. 参考项目

- [Stack-Cairn/LiveAgent](https://github.com/Stack-Cairn/LiveAgent)
- [stablyai/orca](https://github.com/stablyai/orca)

调研时间：2026-07-19。调研用于吸收布局、连接、缓存和移动交互思路，不复制品牌和代码。

## 2. LiveAgent

LiveAgent 提供同一套 React Gateway WebUI，通过媒体查询切换移动侧栏。移动尺寸默认关闭历史侧栏，桌面尺寸默认打开；对话、历史、草稿和工具 Dock 仍共享同一业务状态。

值得借鉴：

- 对话是主页面，空状态使用动态问候和建议卡。
- 建议卡只填充 Prompt，不直接执行。
- 左侧项目/历史在移动端改为覆盖式抽屉。
- 右侧工具 Dock 按需打开，主对话不被工具长期挤压。
- 本地草稿与已持久化会话使用不同 ID，断线不会误写历史。
- 侧栏使用独立 Store 和选择器，活动更新只重绘侧栏子树。
- 连接错误由页面统一展示，子组件不重复堆叠相同错误。
- 浅色/深色使用语义 Token，共用组件代码。

CLI-Manager Web 采用这些模式，但不提供浏览器 xterm；工具执行显示为结构化事件、Diff 和审批。

## 3. Orca

Orca 的移动端不是响应式 Web，而是 Expo Router + React Native 的独立 Companion App。它通过 WebSocket RPC 连接桌面端，按 Host、Worktree、Session 组织路由。

值得借鉴：

- 二维码配对包含端点、设备 Token 和服务端指纹。
- 冷启动和热启动都处理深链接，通知可直接跳转到 Host/Worktree。
- 启动时保留原生 Splash，直到导航树真正完成布局，避免白屏。
- 首页保存最后一次成功快照，冷启动先展示缓存，再原位刷新。
- 仓库列表使用 Host 作用域短期缓存，设置 TTL 和最大条目数。
- Bottom Drawer 处理安全区、键盘高度、拖动关闭和滚动冲突。
- 平板宽屏下限制底部弹层最大宽度，不把手机弹层拉满全屏。
- Right Drawer 根据屏幕宽度计算面板宽度，关闭时不挂载重型手势树。
- 协议版本不兼容时使用阻断页，不让用户进入半可用界面。
- 提供独立 Mock WebSocket Server，移动端开发不依赖真实桌面。

## 4. CLI-Manager 移动 Web 决策

CLI-Manager 首版继续使用响应式 Web/PWA，不开发独立原生 App。吸收 Orca 的移动交互，但使用浏览器能力实现。

| 场景 | Web 实现 |
|---|---|
| 配对 | 扫码或输入短码，服务端转发桌面确认 |
| 首页 | 先展示最后有效缓存，再刷新桌面快照 |
| 导航 | 底部五项导航，深层页面使用浏览器路由 |
| 左侧栏 | 全屏或覆盖式抽屉 |
| 右侧工具 | 全屏抽屉；简单选择使用 Bottom Sheet |
| 键盘 | 使用 `visualViewport` 和安全区变量调整输入框 |
| 安全区 | 使用 `env(safe-area-inset-*)` |
| 通知 | Web Push 点击后跳转到设备、项目和会话 |
| 断线 | 保留只读缓存和草稿，重连后补传事件 |
| 版本不兼容 | 显示阻断页和桌面升级入口 |

## 5. 移动页面层级

```text
首页
├─ 设备选择
├─ 快捷 Prompt
├─ 最近会话
├─ 待审批
└─ 固定输入框

项目
├─ 项目列表
├─ Worktree
├─ 文件
└─ Git / Diff

历史
├─ 会话列表
├─ 会话详情
└─ Replay

我的
├─ 通知
├─ 设备
├─ 主题
└─ 安全
```

## 6. 移动交互规则

- 页面只保留一个主滚动容器。
- 输入框始终位于底部导航上方，并避开虚拟键盘和安全区。
- 抽屉打开后锁定背景滚动，关闭后焦点返回触发按钮。
- Diff、代码和日志进入独立页面，不塞入窄抽屉。
- 危险操作不放在滑动手势唯一入口，必须有明确按钮和确认。
- 缓存内容显示桌面来源和最后上报时间。
- 切换设备、项目或会话时保留各自草稿和滚动位置。

## 7. 不采用的设计

- 不在首版开发 React Native/Expo 应用。
- 不在移动 Web 中运行终端 WebView。
- 不允许手机直接连接桌面局域网 WebSocket；统一经过 Rust Web 服务鉴权和转发。
- 不复制 LiveAgent 或 Orca 的业务导航、品牌资产和原生平台特有实现。
