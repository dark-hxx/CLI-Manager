# 文件与 Git

## 1. 文件浏览器

桌面三栏：文件树、编辑/预览、符号或变更侧栏。支持展开、折叠、刷新、复制路径、下载、搜索、Git 状态和忽略文件分组。`.gitignore` 规则由桌面端解释，Web 只消费结果。

## 2. 文件读取

- 文本文件支持语法高亮、行号、搜索、跳转和换行切换。
- 图片显示尺寸、格式和预览。
- 二进制、大文件、不可读文件显示明确原因，不尝试下载全部内容。
- 文件内容带版本哈希；保存时校验版本，冲突则进入对比流程。

## 3. 文件搜索

支持文件名、路径和内容搜索，筛选项目、Worktree、文件类型、Git 状态和忽略文件。结果展示匹配行、上下文和路径，点击定位。搜索任务可取消，超时显示部分结果。

## 4. 文件写操作

新建、重命名、移动、删除、保存和恢复均由桌面代理执行。覆盖、删除和跨目录移动显示影响路径；用户未授权代码写入时全部禁用。

## 5. Diff

支持 Unified Diff 和 Codex Patch；文件列表、增删行、hunk 导航、行级高亮、折叠未变区、复制片段、跳回触发消息。窄屏进入独立 Diff 页面，代码区域内部横向滚动。

## 6. Git 工作区

页签：变更、暂存、分支、Worktree、提交记录。变更区支持按仓库分组、全部暂存、单文件暂存、撤销暂存、丢弃变更和打开 Diff。

## 7. 分支和提交

- 分支：搜索、切换、新建、删除、查看 ahead/behind。
- 提交：填写信息、选择暂存文件、查看校验结果、提交。
- 推送、强制操作、删除分支属于高风险操作，要求二次确认。
- 多子仓库必须始终显示当前操作目标，禁止跨仓库批量误操作。

## 8. 状态与异常

仓库忙、锁文件、合并冲突、路径缺失、权限不足、文件外部修改、设备离线、操作超时。失败后保留用户输入和当前选择，不自动重复危险操作。

## 9. 移动端

文件树和预览分两级页面；Git 变更使用文件卡片。暂存和提交固定显示选中文件数量，Diff 使用单列 hunk 导航。
