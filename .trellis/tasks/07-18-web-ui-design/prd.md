# Web 端 UI 设计文档与设计稿

## Goal

在不修改产品代码的前提下，为 CLI-Manager Web 端形成覆盖全部桌面能力映射的 UI/UX 设计基线，包括总设计文档、13 份专题文档和主要页面/关键状态高保真原型。

## Requirements

- Changelog Target: `[TEMP]`
- 设计延续现有 CLI-Manager `terminal-green` 深色视觉体系。
- Web 端以结构化 AI 对话替代终端界面。
- 桌面端主工作台采用导航栏、上下文侧栏、主对话区、实时过程栏四区布局。
- 设计覆盖项目、文件、Git、历史、AI Replay、分析、设备和设置的信息架构。
- 输出 SVG 矢量设计稿与 Image 2 高保真 PNG 原型，兼顾后续修改和视觉评审。
- 设计稿必须包含中文真实文案，不使用占位乱码或 Emoji 图标。
- 仅修改设计文档、设计资源、任务记录和 CHANGELOG，不修改 `src/`、`src-tauri/` 或构建配置。
